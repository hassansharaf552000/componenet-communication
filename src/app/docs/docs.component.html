Component Communication in Angular

This page documents major ways Angular components communicate, each with concept, example (TypeScript + HTML), pros/cons, and best use cases.

@Input (Parent → Child)

Concept

Pass data down from a parent to a child using one-way binding.

Example

See live: /examples/input-output. Code: examples/input-output/*

Pros & Cons

  • Pros: Simple, explicit, type-safe.
  • Cons: Only direct parent→child; deep trees need prop drilling.

Best Use Cases

Configure presentational children; pass small view state.

@Output + EventEmitter (Child → Parent)

Concept

Child emits events to notify parent about user actions or changes.

Example

See live: /examples/input-output (@Output section). Code: examples/input-output/user-editor.component.ts

Pros & Cons

  • Pros: Clear contracts; unidirectional data flow.
  • Cons: Only child→parent; cross-sibling requires bubbling.

Best Use Cases

Submit events, selections, notifications from child to parent.

@ViewChild (Parent calls Child)

Concept

Parent gets a reference to a child component/directive and calls methods or reads properties.

Example

See live: /examples/view-child. Code: examples/view-child/*

Pros & Cons

  • Pros: Direct imperative control when needed.
  • Cons: Tighter coupling; lifecycle timing concerns.

Best Use Cases

Focus/scroll, calling child imperative APIs.

Shared Service with RxJS (Subject/BehaviorSubject)

Concept

Singleton service shares state and events across unrelated components.

Example

See live: /examples/shared-service. Code: examples/shared-service/*

Pros & Cons

  • Pros: Decoupled; works across app; testable.
  • Cons: Must manage subscriptions; implicit flows possible.

Best Use Cases

Cross-cutting events; small shared state.

Router Params & Query Params

Concept

Use URL for identity and filters; deep-linkable and shareable.

Example

See live: /examples/router-params/:id. Code: examples/router-params/user-detail.component.ts

Pros & Cons

  • Pros: Bookmarkable; SEO-friendly.
  • Cons: Stringy; payload limits.

Best Use Cases

Resource identity; filter/search/pagination.

Router State (NavigationExtras)

Concept

Pass ephemeral data between routes without polluting URL; not persisted on reload.

Example

See live: /examples/router-state. Code: examples/router-state/*

Pros & Cons

  • Pros: Simple; avoids URL bloat.
  • Cons: Lost on refresh; not shareable.

Best Use Cases

Wizards/checkout handoff.

State Management (NgRx / NGXS / Akita)

Concept

Predictable centralized state with actions/selectors (choose one library).

Example

See live: /examples/state. Code: examples/state/*

Pros & Cons

  • Pros: Scales, tooling, testable.
  • Cons: Boilerplate; learning curve.

Best Use Cases

Medium/large apps; complex shared state.

Angular Signals (v16+)

Concept

Fine-grained reactivity without RxJS; great for local state.

Example

See live: /examples/signals. Code: examples/signals/signals.page.ts

Pros & Cons

  • Pros: Minimal API; performant.
  • Cons: Patterns for app-wide state still evolving.

Best Use Cases

Local component/feature state; computed view state.

Content Projection (ng-content)

Concept

Compose templates by projecting content into slots.

Example

See live: /examples/content-projection. Code: examples/content-projection/*

Pros & Cons

  • Pros: Flexible composition.
  • Cons: No data back unless using inputs/outputs/DI.

Best Use Cases

Cards, layouts, shell components.

Dependency Injection (InjectionToken)

Concept

Provide configuration/abstractions via tokens to decouple consumers.

Example

See live: /examples/injection-token. Code: examples/di/*

Pros & Cons

  • Pros: Replaceable; testable.
  • Cons: Indirection can hide flow.

Best Use Cases

Feature config; strategies/adapters.

Local/Session Storage

Concept

Persist small key/value data in the browser; not reactive.

Example

See live: /examples/storage. Code: examples/storage/*

Pros & Cons

  • Pros: Survives reloads.
  • Cons: Size limits; serialization; security.

Best Use Cases

Preferences; small caches.

RxJS Event Bus Pattern

Concept

Pub/sub with a Subject to broadcast events across the app.

Example

See live: /examples/event-bus. Code: examples/event-bus/*

Pros & Cons

  • Pros: Decoupled; multi-listener.
  • Cons: Harder to trace; leaks if unmanaged.

Best Use Cases

Toasts, modals, cross-feature notifications.
